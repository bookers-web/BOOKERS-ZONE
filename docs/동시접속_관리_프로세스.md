# Bookers Zone 동시접속 관리 프로세스

> 최종 업데이트: 2026-03-04

---

## 1. 전체 흐름도

```
[사용자 브라우저]
       │
       ▼
  ① /login 접속
       │
       ▼
  ② "비회원 관내 이용자" 클릭 → GET /zone/enter
       │
       ▼
  ③ PageController.zoneEnter()
       │
       ├─ IpAuthService.findConfigByIp(clientIp)
       │     └─ TBL_IP_AUTH_CONFIG에서 IP 매칭 (is_enabled=true)
       │     └─ 실패 → www.bookers.life로 리다이렉트
       │
       ├─ IpAuthService.createSession(uisCode, clientIp, userAgent)
       │     ├─ SELECT ... FOR UPDATE (비관적 락)
       │     ├─ 만료 세션 정리 (last_active_at < now - 5분)
       │     ├─ countActiveByUisCode → 현재 접속 수
       │     ├─ 현재 ≥ maxConcurrent → ❌ 실패 (redirect:/login?error=max_concurrent)
       │     └─ 현재 < maxConcurrent → ✅ UUID 세션 생성 → TBL_IP_AUTH_SESSION INSERT
       │
       ├─ 쿠키 설정: BKS_ZONE_SESSION = {sessionId}, maxAge=300초
       │
       ├─ HttpSession 저장: UM_UIS_CODE, UIS_UCP_CODE, ZONE_SESSION_ID, UM_CODE
       │
       └─ redirect:/front/home/main.do
              │
              ▼
  ④ 이후 모든 요청 → IpAuthInterceptor.preHandle()
       │
       ├─ findConfigByIp(clientIp) → 실패 시 www.bookers.life 리다이렉트
       │
       ├─ 쿠키에서 BKS_ZONE_SESSION 추출
       │     ├─ 있음 → validateSession(sessionId, clientIp)
       │     │     ├─ DB에서 세션 조회
       │     │     ├─ last_active_at < now - 5분 → 만료
       │     │     ├─ clientIp 불일치 → 거부
       │     │     └─ 유효 → refreshSession() (last_active_at = NOW()) → ✅ 통과
       │     └─ 없음 → /login 또는 /zone/enter면 통과, 아니면 /login으로 리다이렉트
       │
       ▼
  ⑤ 스케줄러 (1분마다)
       └─ SessionCleanupScheduler → cleanupExpiredSessions()
             └─ DELETE FROM TBL_IP_AUTH_SESSION WHERE last_active_at < now - 5분
```

## 2. 핵심 설정값

| 항목 | 값 | 위치 |
|------|-----|------|
| 세션 타임아웃 | **5분** | `IpAuthServiceImpl.SESSION_TIMEOUT_MINUTES = 5` |
| 쿠키 만료 | **5분 (300초)** | `PageController.COOKIE_MAX_AGE = 60 * 5` |
| 쿠키 이름 | `BKS_ZONE_SESSION` | `IpAuthInterceptor`, `PageController` |
| 최대 동시접속 수 | **DB에서 기관별 설정** | `TBL_IP_AUTH_CONFIG.max_concurrent` |
| 만료 세션 정리 주기 | **1분마다** | `@Scheduled(fixedRate = 60000)` |
| 인터셉터 적용 범위 | `/**` (전체) | `WebConfig.addInterceptors()` |
| 인터셉터 제외 | `/api/ip-auth/**`, `/api/guest/**`, `/error/**`, 정적 리소스 | 동일 |

## 3. DB 테이블 구조

### TBL_IP_AUTH_CONFIG (기관별 IP 인증 설정)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INT (PK, auto) | 자동 증가 ID |
| `uis_code` | VARCHAR | 기관 코드 |
| `uis_name` | VARCHAR | 기관명 |
| `ip_version` | VARCHAR | IPv4/IPv6 |
| `ip_pattern` | VARCHAR | 허용 IP (쉼표 구분) |
| `max_concurrent` | INT | **최대 동시접속 수** |
| `is_enabled` | BOOLEAN | 활성화 여부 |
| `created_at` | DATETIME | 생성 시각 |
| `updated_at` | DATETIME | 수정 시각 |

### TBL_IP_AUTH_SESSION (현재 활성 세션)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `session_id` | VARCHAR (PK) | UUID |
| `uis_code` | VARCHAR | 기관 코드 |
| `client_ip` | VARCHAR | 클라이언트 IP |
| `user_agent` | VARCHAR | 브라우저 UA |
| `um_code` | VARCHAR | Zone 게스트 회원코드 |
| `created_at` | DATETIME | 생성 시각 |
| `last_active_at` | DATETIME | **마지막 활동 시각** (갱신 기준) |

## 4. 동시접속 제어 핵심 로직 (3중 방어)

### 4.1 세션 생성 시 (진입 시점)

```java
// IpAuthServiceImpl.createSession()
config = configMapper.selectByUisCodeForUpdate(uisCode);  // FOR UPDATE 비관적 락
// → 만료 세션 정리
// → countActiveByUisCode → 현재 수
// → 현재 >= maxConcurrent → 거부
// → 미만 → UUID 세션 생성 + INSERT
```

- `SELECT ... FOR UPDATE`로 동시 요청 시 race condition 방지
- 세션 생성 전 만료 세션을 먼저 정리하여 정확한 카운트 보장

### 4.2 매 요청마다 (인터셉터)

```java
// IpAuthInterceptor.preHandle()
// 쿠키에서 sessionId 추출 → validateSession() → refreshSession()
// → last_active_at을 NOW()로 갱신 (5분 타이머 리셋)
```

- 활동이 있으면 세션 유지, 5분간 활동 없으면 자동 만료
- IP 바인딩 체크: 세션 생성 시 IP와 현재 IP 불일치 시 거부

### 4.3 백그라운드 정리 (스케줄러)

```java
// 1분마다 실행
DELETE FROM TBL_IP_AUTH_SESSION WHERE last_active_at < NOW() - 5분
```

- 브라우저 닫기, 네트워크 끊김 등 명시적 로그아웃 없이 이탈한 세션 정리
- 동시접속 슬롯이 좀비 세션에 의해 점유되는 것을 방지

## 5. 세션 생명주기

```
[생성] /zone/enter → UUID 발급 → DB INSERT + 쿠키
  ↓
[유지] 매 요청 → 인터셉터에서 last_active_at 갱신 (5분 리셋)
  ↓
[만료] 5분간 활동 없음 → validateSession()에서 만료 판정
  ↓                       OR
[정리] 스케줄러가 1분마다 만료 세션 DELETE
  ↓                       OR
[강제] destroySession(sessionId) → 즉시 DELETE
```

## 7. 주의사항

1. **FOR UPDATE 락**: `createSession()` 시 `selectByUisCodeForUpdate`로 비관적 락을 잡으므로, 동시 진입 요청 시 직렬화됨
2. **이중 만료 체크**: 인터셉터(validateSession)와 스케줄러(cleanupExpiredSessions) 양쪽에서 만료를 체크하므로, 스케줄러 실행 전이라도 만료된 세션은 즉시 거부됨
3. **쿠키-DB 동기화**: 쿠키 maxAge(5분)와 DB 세션 타임아웃(5분)이 동일하므로, 쿠키 만료 시 DB 세션도 자연 만료됨
4. **IP 바인딩**: 세션은 생성 시 IP에 바인딩되어, 다른 IP에서 같은 세션 쿠키를 사용할 수 없음
